# 05-测试与验证策略

## 一、验证目标

每轮至少证明三件事：

1. 核心功能逻辑正确
2. 工程状态可运行 / 可构建
3. 本轮没有明显回归

## 二、推荐验证层级

### 1. 快速验证
- lint
- typecheck
- build

### 2. 功能验证
- 单元测试
- 集成测试
- 接口测试

### 3. 页面 / 体验验证
- 浏览器手动检查
- Playwright 或同类自动化验证
- 关键页面截图留证

## 三、本项目待补充命令

- `lint`: 待补充
- `typecheck`: 待补充
- `test`: 待补充
- `build`: 待补充
- `e2e`: 待补充

## 四、执行规则

1. 修改前先判断本轮需要覆盖的验证范围
2. 修改后必须至少运行与改动相关的验证
3. 声称“完成”前必须读取实际输出
4. 验证结果要回写到迭代文档

## 五、发布前最小验证集合

- 构建通过
- 关键路径验证通过
- 无阻塞性错误
- 部署步骤至少走通一次
