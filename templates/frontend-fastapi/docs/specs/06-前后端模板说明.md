# 06-前后端模板说明

## 当前模板

- 模板类型：`frontend-fastapi`

## 适用场景

- 前端 + Python 后端
- API 服务驱动的 AI 工具
- 需要前后端最小闭环的项目

## Python 环境策略

优先级如下：

1. `conda`
2. `uv`
3. 都没有时写为 `待确认`

本模板不再提供默认环境初始化脚本。

原因：目标项目的 Python 版本、包管理器、部署方式和本地环境差异很大，应该由后端 Agent 先确认实际约束，再把最终命令写入目标项目 runbook。

## 日志要求

至少保留以下日志：

1. 应用启动日志
2. 请求开始与结束日志
3. 关键流程日志
4. 异常日志

日志不是越多越好，但关键流程必须能追踪。

## 注释要求

注释保持简短即可，但这些位置必须可读：

- 配置
- 初始化
- 边界判断
- 关键服务入口
