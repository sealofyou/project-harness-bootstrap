# 00-提示词索引

本目录保存本项目默认可复用的 Agent 提示词资产。

这些提示词不是一次性全文塞给 Agent 的长上下文，而是按任务阶段选择调用的窄职责模块。系统提示词负责调度，具体提示词负责完成单一类型的工作。

## 默认链路

1. `01-系统级规则.md`
2. `10-系统-默认协作流程.md`
3. `05-启动后交互契约.md`
4. `20-产品-需求深访与PRD.md`
5. `30-前端-设计与实现.md`
6. `31-前端-设计系统与风格.md`
7. `40-后端-接口与数据.md`
8. `50-Git-仓库维护.md`
9. `60-测试-交互视觉与验收.md`
10. `70-协作-远程仓库与Linear.md`

## 调用原则

- 系统提示词只做分流、约束和完成检查。
- 系统级规则提示词对所有任务生效，优先于专门提示词。
- 启动后交互提示词负责项目刚初始化后的第一轮提问和交接。
- 产品提示词只负责澄清需求、生成 PRD、维护需求事实。
- 前端提示词只负责界面、交互、视觉和前端实现，不擅自改业务边界。
- 前端设计系统提示词必须在前端代码实现前使用，用来确认风格、页面结构、响应式和视觉验收标准。
- 后端提示词只负责接口、数据、服务边界，不擅自补产品规则。
- Git 提示词只负责分支、提交、PR、可追溯记录。
- 测试提示词覆盖交互、回归、普通用户体验、视觉美观度和设计达标验收。
- 远程仓库 / Linear / 多人协作是可选协作层，按项目需要接入，不替代 repo docs。

## 后续扩展

如果某一类提示词变长，优先拆成更细的文件，而不是继续扩大单个提示词。例如：

- `21-产品-用户旅程.md`
- `22-产品-故事卡.md`
- `31-前端-设计系统.md`
- `32-前端-页面实现.md`
- `61-测试-普通用户体验.md`
- `62-测试-视觉美观度.md`
