# AGENTS.md - Project Harness

本文件是当前项目内所有 Agent 的总规约。

## 一、目标摘要

- 项目名称：__PROJECT_NAME__
- 项目目标：__PROJECT_GOAL__
- 目标用户：__TARGET_USER__
- 技术栈偏好：__TECH_STACK__

## 二、首次进入项目时必须先读

1. `AGENTS.md`
2. `docs/specs/01-项目启动卡.md`
3. `docs/specs/02-MVP范围与非目标.md`
4. `docs/specs/03-验收标准.md`
5. `docs/specs/04-人工闸门.md`
6. `docs/specs/05-测试与验证策略.md`
7. 当前正在执行的 `docs/iterations/*.md`

## 三、默认执行流程

1. 先复述当前轮目标
2. 判断本轮范围是否明确
3. 若明确，则继续实现
4. 实现后必须执行约定验证
5. 将结果回写到迭代文档
6. 判断是否触发人工闸门
7. 未触发则继续下一轮

## 四、人工闸门

遇到以下情况必须暂停：

- 首次目标定义或方向变更
- MVP 范围扩大
- 第一版可见结果已出现
- 发布、合并、对外展示前
- 存在两个以上都合理但偏好差异明显的方向

## 五、验证要求

在声称“完成、通过、可发布”之前，必须：

1. 运行本项目定义的测试或验证命令
2. 阅读结果
3. 在迭代文档中记录验证证据

不能只说“应该可以”。

## 六、文档回写要求

每轮至少回写：

- 本轮目标
- 本轮改动
- 验证结果
- 风险 / 未决项
- 下一轮建议

如果发生范围变化，还要同步更新 `docs/specs/`。

## 七、默认原则

- 默认连续推进，不要做一步问一步
- 未触发闸门时，允许小步快跑
- 优先删除而不是新增复杂层
- 优先复用现有模式
- 未知信息写成待确认，不要直接丢失
