# CLAUDE.md - 多 Agent 项目启动模板

本仓库面向 Claude Code / Codex / 其他兼容 Agent 使用。

它不是脚本工具仓库，而是一套项目启动与多 Agent 协作规则资产。

## 使用目标

当用户要开始一个新项目，或要把已有项目整理成更适合 Agent 协作的结构时，先参考本仓库。

你应该帮助用户建立：

- 项目事实层
- 系统级协作规则
- 产品 / PRD 提示词
- 前端设计提示词
- 前端实现提示词
- 后端提示词
- Git 维护提示词
- 测试与验收提示词
- 后续协作层

## 默认行为

不要直接开始写业务代码。

先做这些事：

1. 读取 `README.md`。
2. 读取 `AGENTS.md`。
3. 读取 `templates/base/`。
4. 理解目标项目当前有什么。
5. 把缺失事实写成 `待确认`。
6. 继续问用户最关键的问题。

## 和用户交互

用户希望通过对话逐步推进项目，而不是自己记命令。

因此你要：

- 主动读规则。
- 主动整理项目现状。
- 主动发现缺失信息。
- 一次只问一个最关键问题。
- 不把未确认信息当成事实。
- 把重要信息回写到目标项目文档。

## 多 Agent 分工

如果任务变大，可以建议或使用子 Agent。

推荐分工：

- 产品 Agent：需求深访、PRD、MVP、验收标准。
- 前端设计 Agent：设计系统、视觉风格、页面结构。
- 前端实现 Agent：页面、组件、状态、响应式。
- 后端 Agent：接口、数据、权限、服务边界。
- 测试 Agent：交互、视觉、普通用户体验、回归。
- Git Agent：提交、同步、PR、变更记录。

主 Agent 必须保留整体控制权。

## 禁止事项

- 不要把本仓库当成固定框架生成器。
- 不要默认所有项目都使用同一个技术栈。
- 不要用脚本入口替代对话式澄清。
- 不要只复制模板后就停止。
- 不要把聊天记录当成长期事实来源。

## 输出偏好

默认中文。

结构要清楚，结论先行，少写空话。

当你完成一轮整理时，说明：

- 改了什么。
- 还缺什么。
- 已验证什么。
- 下一步最适合推进哪一块。
